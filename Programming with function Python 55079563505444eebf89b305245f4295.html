Programming with function: Python

In this course Programming with function: Python you will learn: what is a function, how does function works, the types of function, the calling function and called function, and at last formal & actual parameters.

A program is a set of statements that takes some input, does specific computations based on the given input, and produces the desired output. A very large program with a huge single list of instructions increases complexity. So Python allows us to divide a large program into some small independent units or blocks known as functions. Decomposing a complex problem into a simpler one using functions improves the clarity of the code.

Functions are the most important segments of subprograms of an application used to perform specific tasks. A python program can have one or more functions.

Advantages of using Python

  • Induce reusability of code.
    • We can call python functions any number of times from any part of the program. So function induces reusability in a program.
  • Reduce duplication of code.
    • By using functions, we can avoid rewriting the same logic/code, again and again, Thus function reduces program size.

Types of Functions

Basically, there are three types of functions used in the python program:

  • Built-in function (python library function)
    • These are predefined functions and are always available in the python library
  • The function is defined within modules.
    • These are also predefined functions available in different modules.
  • User-defined function
    • These are defined by the user/programmer/us.

How does a function work?

  • Execution always begins from the first statement of the program.
  • A python program may contain several function definitions.
  • If any function definition is found, python executes only the function header for its correctness of it and skips all lines of the function body(block).
  • When python sequentially reaches the top-level statement‚Äôs function call, python transfers control to the function header and then execution of the function body takes place.
  • Finally function execution ends with a return statement if any or the last statement of the function body.

The flow of execution in a function call

  • The flow of execution refers to the order in which statements are executed.
  • A function body is executed in the execution frame.
  • Whenever a function call statement is executed, the execution frame for the called function is created and the control is transferred to invoke the called function.
  • Within the function‚Äôs execution frame, the body of the function gets executed and after the last statement of the function, the control returns to the statement with/without any value(s) to the function from where it is called(calling function).

Function Parameters

The values being passed through a function call statement are called arguments or actual parameters. The values received in the function definition are called parameters or formal parameters.

A function has two types of parameters

  • Formal parameter(parameters):
    • Formal parameters are written in the function prototype(function definition). Formal parameters are local variables that are assigned values from the arguments when the function is called.
  • Actual parameter(arguments):
    • When a function is called, the values that are passed are called actual parameters. At the time of the call, each actual parameter is assigned to the corresponding formal parameter in the function definition.

Lambada function

Python lambda function doesn‚Äôt have any return statement. It has only a single expression which is always returned by default. The Python lambda function is anonymous as it is a function without a def keyword and name. To create a Python lambda function, we have to use the lambda keyword.

The basic syntax of python lambda is:

Lambda arguments : expression

The Python lambda function accepts any number of arguments but uses only one expression.

For instance, lambda a, b: a + b. Here, a and b are the arguments accepted by the lambda function. a + b is the expression.

add = lambda x, y : x + y
print(add(10, 20))

print("\nResult from a Function")
def add_func(x, y):
		return x + y

print(add_func(10, 20))

Both the lambda function and the regular function return the same result. However, the regular function needs a def keyword, function name, and a return value. Whereas, the lambda function does not need any of them. By default, it returns the expression result.

>>> f = lambda x,y : 2 * x + y
>>> f
<function <lambda> at 0x87d30>
>>> f(3, 4)
10
>>> v = lambda x: x*x(100)
>>> v
<function <lambda> at 0x87df0>
>>> v = (lambda x: x*x)(100)
>>> v
10000
